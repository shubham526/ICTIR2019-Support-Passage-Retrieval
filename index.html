Abstract

Our goal is to complement an entity ranking with human-readable explanations of how those retrieved entities are connected to the information need. While related to the problem of support passage retrieval, in this paper, we explore two underutilized indicators of relevance: contextual entities and entity salience. The effectiveness of the indicators are studied within a supervised learning-to-rank framework on a dataset from TREC Complex Answer Retrieval. We find that salience is a useful indicator, but it is often not applicable. In contrast, although performance improvements are obtained by using contextual entities, using contextual words still outperforms contextual entities.

Background: Entity Salience

We want passages in which the entity is salient. By salient, we mean that the entity is central to the passage. For example, see Section 1 in the paper.

To find salience entities in a passage, we use the API SWAT . For more details on the API, you may read the paper here and see the API documentation here.

Approach

We describe the overarching approach in detail below. All the features are derived from some variation of this approach. For more details on individual features, please refer to the paper.

Overview

The Problem. Consider the query Brexit. An entity relevant to this query may be Theresa May. We want a paragraph which would explain to the user, why the entity Theresa May is relevant to the query Brexit. So what we want is a three-way relevance: paragraph to query, paragraph to entity and paragraph to query-entity pair. We can find relevant paragraphs and entities for a query using any standard information retrieval model such as BM25. The trick is to model the relevance of the paragraph for both the query and the entity.

The Solution. Long time ago, Page et al. came up with an algorithm called PageRank to rank web pages on the internet. The crux of their method was: good pages point to other good pages. So even if a page has fewer inlinks, its PageRank may be higher than another page which has more inlinks because it is pointed to by a more important page. Analogously, we may say that a paragraph with a lot of relevant (according to retrieved result) entities for the query which co-occur with a given entity is more relevant to that entity than any other paragraph. In other words, if we know the most frequently co-occurring entities with another entity, and a paragraph contains a lot of these entities, then that paragraph is very much relevant for the entity, perhaps more than any other. As an example, for the query Brexit, some relevant entities may be Theresa May, UK and David Cameron. Suppose we want a support passage for the entity Theresa May. If we know that the entities UK and David Cameron co-occur most frequently with the entity Theresa May and we find a paragraph which contains these entities, then it is likely talking about Theresa May as well. If we couple this with the knowledge that the paragraph is also relevant for the query, then we get exactly what we need to define a support passage for the query-entity pair. We would have then solved the problem!

Entity Context Document(ECD). The question then arises: how do we find the most frequently occurring entities with a given entity? Our approach is: For every entity relevant to the query, find a paragraph (relevant to the query) which mentions this entity and "stitch" them all together into one document about this entity. This document, which we call an Entity Context Document, and denote as \( D_e \), is then a concise document that tells us everything relevant about this entity and the query. All the entities which occur in this Entity Context Document are the entities that co-occur with the given entity (the entity the Entity Context Document is about). Now we know all the co-occurring entities with a given entity. Note that when we say entity relevant to a query above, we mean relevant according to the entity ranking provided to us. This is shown in figure below.

The Model. We propose the following model: the probability of seeing an entity \( e'\) given that we have already seen the entity \( e\) and the query \( q\) is the sum of the frequency of the entity \( e'\) , summed over all \( p \in D_e \). We say that a paragraph is relevant for the query-entity pair, if it is relevant for the query and it mentions the entity (contains a link to this entity). This can be written more formally using conditional probabilities as \[ P(e' \vert e, q) \propto count(e') \tag{1}\] where \(p\) is a paragraph, \( D_e \) is the ECD about entity \( e \), \( e' (\ne e) \) is an entity co-occurring with \( e \), and \(count(e')\) is the number of times \( e' \) is mentioned in \( p \).

Equation 1 gives us a distribution over co-occurring entities for a given entity. It tells us how likely we are to see an entity \( e' \) provided we have already seen the query-entity pair \( (q, e) \). It gives us an intuition about which entities co-occur most frequently with a given entity. Knowing these entities, we are now in a position to derive features which we can use to train a machine learning model.
